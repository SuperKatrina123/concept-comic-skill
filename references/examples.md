# Examples

## Highest Priority

### Input

```yaml
rawIdea: Highest Priority
mode: literal
domain: office
tone: deadpan
platformTarget: comic_only
```

### Why This Mode Works

`Highest Priority` is strongest when the phrase is treated as a physical instruction. The joke dies if the model upgrades it into a life lesson about rest or values.

### Angle Candidates

1. `放到最高处`
   Premise: 经理说任务是 high priority，员工把文件夹放到办公室最高的架子上。
   Why: 直接、清楚、最后一格容易补刀。
2. `优先通道`
   Premise: 员工给文件做了一个 VIP 电梯通道，优先升到天花板附近。
   Why: 画面更夸张，但比第一个更绕。
3. `高层汇报`
   Premise: 员工坚持把任务交给楼里最高楼层的人处理。
   Why: 也能成立，但动作没有第一个干净。

### Chosen Breakdown

- `title`: 最高优先级
- `coreJoke`: 把 high priority 按物理高度理解
- `normalWorld`: 经理只是在说这件事很重要，要先处理
- `twist`: 员工认真执行“高”这个字面含义
- `finalPayoff`: 员工看着最顶层架子说“Done. Highest priority.”

### Storyboard

1. Panel 1, setup
   Scene: 办公室里，经理把写着 URGENT 的文件夹递给员工。
   Action: 经理认真交代任务。
   Dialogue: “This task is high priority.”
2. Panel 2, trigger
   Scene: 员工点头，环顾办公室，视线停在最高的架子。
   Action: 员工认真理解指令。
   Dialogue: “明白。”
3. Panel 3, twist
   Scene: 员工踩着梯子，把文件夹放到办公室最顶上的架子。
   Action: 文件被举到几乎碰天花板的位置。
   Dialogue: 无
4. Panel 4, payoff
   Scene: 员工从梯子上下来，经理抬头看着高处的文件夹。
   Action: 员工平静汇报完成。
   Dialogue: “Done. Highest priority.”

## Open Office

### Input

```yaml
rawIdea: Open Office
mode: satire
domain: office
tone: deadpan
platformTarget: comic_only
```

### Why This Mode Works

字面梗可以做，但最强的版本是讽刺“开放协作”的理想叙事，如何一步步逼得员工自己搭边界。

### Angle Candidates

1. `自建围挡`
   Premise: 公司宣布更开放，员工开始用耳机、绿植、纸板和显示器搭私有领地。
   Why: 现实感强，结尾容易反讽。
2. `开放麦`
   Premise: 所有人都被迫开放交流，最后变成谁都在互相打断。
   Why: 能做，但画面辨识度略低。
3. `透明管理`
   Premise: 老板说透明沟通，员工把自己包进透明隔音罩。
   Why: 画面好，但没有第一个稳定。

### Chosen Breakdown

- `title`: 开放办公
- `coreJoke`: 名义上的开放协作，现实里变成每个人都在拼命制造边界
- `normalWorld`: 公司相信 open office 会提高协作效率
- `twist`: 员工开始主动搭出临时隔离带
- `finalPayoff`: 老板看着越来越封闭的空间，说“更开放了”

### Storyboard

1. Panel 1, setup
   Scene: 老板在新办公室前介绍“Open Office”理念。
   Action: 墙上挂着“More Collaboration”标语。
   Dialogue: “开放办公，让我们更协作。”
2. Panel 2, trigger
   Scene: 员工们坐进没有隔断的大空间里，四面八方都有声音。
   Action: 大家表情逐渐僵硬。
   Dialogue: 无
3. Panel 3, twist
   Scene: 员工们开始摆耳机、绿植、纸板、显示器，给自己围出小边界。
   Action: 开放空间被切成一个个小堡垒。
   Dialogue: 无
4. Panel 4, payoff
   Scene: 老板满意地看着这些临时围挡。
   Action: 老板一本正经地下结论。
   Dialogue: “很好，大家现在更开放了。”

## Human in the Loop

### Input

```yaml
rawIdea: Human in the Loop
mode: auto
domain: ai
tone: deadpan
platformTarget: comic_only
```

### Why This Example Is Split

这个词同时适合 `literal` 和 `satire`。第一版 skill 应该展示两条路都成立，但不要把两条路揉成一个结果。

### Literal Path

- Premise: 会上有人说要 keep a human in the loop，工程师真的造了一个环形装置，把人放进去。
- Payoff: 里面的人问“Can I get out now?” 外面的人回答 “No, you’re the loop.”

### Satire Path

- Premise: 大家说 AI 把人解放出来了，但人的工作只是从亲手执行变成了写规则、补 prompt、做校验、处理例外。
- Payoff: 最后一格显示系统自动运行，旁边那个人还在不停接漏出来的任务。

### Preferred Note

如果用户想要“字面梗”，走 literal。
如果用户想表达“人没有退出系统，只是换了劳动形式”，走 satire。

## Model Distillation

### Input

```yaml
rawIdea: Model Distillation
mode: literal
domain: ai
tone: deadpan
platformTarget: comic_only
```

### Why This Mode Works

AI 术语很适合直接物理化。把 distillation 画成蒸馏过程，比解释知识蒸馏原理更好笑也更直观。

### Angle Candidates

1. `蒸出小瓶`
   Premise: 一个巨大的 Large Model 被蒸馏成一小瓶标签写着 Smaller, Faster 的液体。
   Why: 画面直接，包袱稳定。
2. `实验室浓缩`
   Premise: 工程师像化学家一样反复过滤模型，只剩一个便携版本。
   Why: 能成立，但没有第一个干净。
3. `试喝评测`
   Premise: 团队把蒸出来的小模型当饮料品鉴。
   Why: 可以给第 4 格 deadpan 台词。

### Chosen Breakdown

- `title`: 模型蒸馏
- `coreJoke`: 把 distillation 按厨房/实验室蒸馏来画
- `normalWorld`: model distillation 指把大模型知识压到更小模型里
- `twist`: 团队真的在蒸一个写着 Large Model 的大桶
- `finalPayoff`: 有人尝一口小瓶，说 “Still weird.”

### Storyboard

1. Panel 1, setup
   Scene: 实验室里放着一个写着 Large Model 的巨大金属桶。
   Action: 工程师准备设备。
   Dialogue: “We’re doing model distillation.”
2. Panel 2, trigger
   Scene: 管子、冷凝器和小瓶子都接好。
   Action: 工程师按下开关，认真开始蒸。
   Dialogue: “Okay.”
3. Panel 3, twist
   Scene: 一滴滴液体流进一只写着 Smaller, Faster 的小瓶。
   Action: 大桶明显缩了很多。
   Dialogue: 无
4. Panel 4, payoff
   Scene: 同事拿起小瓶尝了一口。
   Action: 所有人认真等待评价。
   Dialogue: “Still weird.”

## Guardrails

### Input

```yaml
rawIdea: guardrails
mode: auto
domain: ai
tone: deadpan
platformTarget: comic_only
```

### Why This Mode Works

`guardrails` 默认最适合走 `literal`，因为这个词本身已经是一个强物理对象。直接把 AI safety 里的 guardrails 画成真的公路护栏，画面简洁、语义清楚，而且很适合 deadpan 结尾。

### Angle Candidates

1. `给模型装护栏`
   Premise: 团队说要给 AI 加 guardrails，工程师真的给聊天机器人电脑围上公路护栏。
   Why: 最直接，视觉一眼懂，最后一格稳定。
2. `回复单车道`
   Premise: 工程师把模型输出路径做成一条两边全是护栏的狭窄车道。
   Why: 也成立，但更概念化一些。
3. `层层加栏`
   Premise: 每次评审都要求“再多一点 guardrails”，最后电脑被一圈圈护栏围到几乎不能动。
   Why: 可以升级成 satire，但默认 literal 没第一个干净。

### Chosen Breakdown

- `title`: 给模型加护栏
- `coreJoke`: 把 AI guardrails 按真实护栏来执行
- `normalWorld`: 团队本意是给模型增加边界和限制，减少越界输出
- `twist`: 工程师认真按字面理解，在电脑周围装上沉重的金属护栏
- `finalPayoff`: 大家满意地确认它现在不会越界了，同时也几乎没法动了

### Storyboard

1. Panel 1, setup
   Scene: AI 团队会议室，屏幕上写着 “Need stronger guardrails.”
   Action: PM 指着屏幕提需求。
   Dialogue: “We need stronger guardrails.”
2. Panel 2, trigger
   Scene: 工程师站起来记笔记，旁边已经打开施工用品目录。
   Action: 他非常认真地接受任务。
   Dialogue: “Understood.”
3. Panel 3, twist
   Scene: 办公区里，一台显示聊天界面的电脑被银色公路护栏围起来，像一个微型施工现场。
   Action: 工程师还在拧最后一颗螺丝。
   Dialogue: 无
4. Panel 4, payoff
   Scene: PM 和工程师站在护栏外，平静看着里面那台被困住的电脑。
   Action: PM 满意地点头，AI 对话框被挤在栏内一小块空间里。
   Dialogue: “Perfect. It can’t go off track now.”
